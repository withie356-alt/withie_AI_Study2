Part 1

52g는 어떻게 돌아가는가

1. 52g Crew

각 계열사에서 참여하는 현업 구성원으로, 현장의 문제 해결 프로젝트를 리딩하는 핵심 인력

이런 마음가짐이라면 지원하세요

변화를 즐기는 유연함 현장에서 출발하는 공감 함께 만드는 즐거움 동료와 함께 하는 성장 작고 빠른 실행 믿음이 가는 탁월함

2. 역량 강화 트레이닝

  • 크루서블 스프린트 — 다양한 회사의 Crew가 한 공간에 모여 디자인씽킹과 GenAI로 문제를 직접 해결하는 집중 합동근무
  • 레슨런/사례 공유 — Crew의 프로젝트 경험을 내부 공유
  • 외부 강사/타사 사례 — 외부 전문가와 타사 혁신 사례 학습
  • 튼튼쑥쑥 — 정기 성장 프로그램으로 학습 리듬 형성
  • 해커톤 — MISO, v0 등으로 직접 만들며 도구 활용력과 협업을 체득하는 실전 훈련

3. 프로젝트 진행

Crew가 현업 구성원들과 팀을 구성해 현장의 실제 문제를 해결합니다. 프로젝트에는 Crew와는 별도로 52g 전담 구성원이 함께합니다.

Catalyst(과제 오너) — 문제를 과제로 만들고, 이해관계자를 정렬해 끝까지 실행하며 확산·제품화로 연결

Facilitator(과정 설계자) — 워크숍을 설계·진행하고, 논의 흐름을 유지하며 시각화로 합의를 촉진

여러분도 Crew가 될 수 있다!
Part 2

디자인씽킹이란?

52g가 현장의 문제를 풀 때 사용하는 핵심 방법론

사람(사용자)을 깊이 이해해 ‘진짜 문제’를 정의하고,
팀 협업으로 빠르게 프로토타입을 만들고 테스트하며 개선을 반복해 해법을 찾는 방식

어디서 왔는가

  • • 1957년부터 스탠퍼드에서 학제 간 협업으로 발전
  • • 2005년 Stanford d.school 설립으로 교육 모델로 확산
  • • IDEO의 Tim Brown, David Kelley가 산업계에 대중화

Auernhammer & Roth, 2021

핵심 원리

  • • 고객(사용자)에 대한 깊은 이해(공감)에서 출발
  • • 관찰·인터뷰 등으로 니즈를 발견
  • • 아이디어를 프로토타입으로 구체화하고 빠르게 테스트
  • • 인간 중심(human-centered) 철학에 기반
Part 2

디자인씽킹 5단계 프로세스

1. 공감

Empathize
고객의 시선으로 보기

→

2. 정의

Define
진짜 문제 명확히

→

3. 아이데이션

Ideate
아이디어를 최대한 발산

→

4. 프로토타입

Prototype
최소 비용으로 제작

→

5. 테스트

Test
피드백으로 개선

이 과정은 직선이 아니라 반복(iterate)입니다. 각 단계에서 발산과 수렴을 오가며 반복하는 과정입니다.

Example

디자인씽킹 실전 예시

“신입사원 온보딩 교육, 현장에 가면 다 까먹는다”

1. 문제발견
온보딩 설문 서술형 답변을 담당자가 하나하나 읽음 → “교육은 많은데 현장에서 기억이 안 나요”
2. 문제정의
배운 건 기억나는데, 구체적인 순서와 세부 내용이 현장에서 안 떠오른다
3. 아이데이션
현장 OJT(현장실습) 강화, 요약 핸드북, 멘토링 확대, 교육 영상 제작… 아이디어 발산
4. 프로토타입
핵심 절차만 추린 ‘현장용 요약 핸드북’ 시범 제작
5. 테스트
“핸드북도 결국 안 펴봐요, 상황마다 다른데 하나로는 부족해요” → 3단계로 복귀

처음부터 완벽한 답이 아니라, 빠르게 만들고 → 피드백 받고 → 다시 돌아가는 반복

AI + Example

같은 문제, AI와 함께라면?

온보딩 설문 데이터 + AI = 더 빠른 문제 발견과 해결

기존
AI와 함께
1. 문제발견
⏱ 2~3주설문 답변을 담당자가 하나하나 읽음
“현장에서 기억이 안 나요”
⚡ 5분설문 서술형 답변 전체를 AI에 입력
→ “현장 적용” 관련 불만 45%로 최다
2. 문제정의
⏱ 회의 수 회배운 건 기억나는데
순서·세부 내용이 안 떠오른다
⚡ 10분1단계 설문 데이터를 지식으로 가져온 뒤
프롬프트: “신입사원 입장에서 현장의 어려움을 알려줘”
→ “대충은 아는데 순서랑 세부 절차가 안 떠올라요”
3. 아이데이션
⏱ 반나절현장 OJT(현장실습) 강화, 요약 핸드북,
멘토링 확대… 발산
⚡ 5분AI에게 “현장에서 바로 도움받을 방법” 요청
교육 내용 챗봇, 상황별 가이드 등 발산
4. 프로토타입
⏱ 1~2주핵심 절차만 추린
‘현장용 요약 핸드북’
⚡ 10분v0로 온보딩 도우미 챗봇 제작
교육 내용 기반으로 바로 답변
5. 테스트
⏱ 다시 몇 주“핸드북도 결국 안 펴봐요”
→ 3단계로 복귀
⚡ 즉시“답변이 너무 길어” → AI에게 즉시 수정
4 ↔ 5단계 빠른 반복

AI가 빠른 인사이트를 주지만, 사람과 공감하고 최종 판단을 내리는 건 사람의 몫입니다

AI Design Thinking

AI 디자인씽킹 — 단계를 상황에 맞게

AI와 함께할 때, 5단계를 반드시 순서대로 밟을 필요는 없습니다

단계 압축

공감 + 정의를 AI 분석으로 동시에 수행

데이터가 충분할 때

단계 건너뛰기

문제가 명확하면 바로 프로토타입부터 시작

빠른 검증이 필요할 때

단계 반복

테스트 결과로 어느 단계든 즉시 회귀

AI가 반복 비용을 낮춰줌

5단계는 “체크리스트”가 아니라 “도구 상자”입니다.
상황에 맞는 도구를 꺼내 쓰세요.

Part 3

GenAI가 디자인씽킹을 바꾸고 있다

GenAI(Generative AI) — 텍스트·이미지·코드 등을 스스로 생성할 수 있는 AI

기존 (Traditional Design)

사람이 사용자의 문제에 공감
↓
사람이 문제 발견·정의·해결

지금 (Generative Design)

사람이 사용자의 문제에 공감
↓
AI와 함께 문제 발견·정의·해결을 빠르게

사람 + AI 협업의 핵심: AI가 아이디어를 제안하고, 사람이 비판적으로 평가하고, 다시 반복합니다.
최종 판단은 언제나 사람의 몫입니다.

AI를 쓸 줄 아는 것 ≠ AI를 잘 쓰는 것. 결과를 비판적으로 평가하고 방향을 잡는 것이 진짜 역량
